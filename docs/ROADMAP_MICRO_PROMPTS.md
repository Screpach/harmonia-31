# Roadmap: Micro Prompts

This roadmap sequences small, reviewable prompts while private rule data is unavailable.

## Phase 0: planning and contracts

1. ✅ Repository charter and agent instructions.
2. ✅ Define initial TypeScript workspace and package boundaries.
3. ✅ Add domain primitives for 31-EDO spelled pitch identity.
4. Add adapter contracts for rendering/audio/storage.
5. Integration gate: verify docs, contracts, and build integrity only.

## Phase 1: synthetic rule scaffolding

1. Add `awaiting-private-rule-pack` fixture contracts.
2. Implement non-historical mechanical invariant checks.
3. Add SATB voice container types and validation surface.
4. Add deterministic test fixtures for harmony states.
5. Integration gate: no new features; strengthen tests and docs.

## Open-source research direction

Future prompts may evaluate permissive-license libraries for:
- audio playback,
- score-like rendering,
- state management,
- property-based testing.

Any adoption proposal must document license compatibility and adapter-boundary fit.
